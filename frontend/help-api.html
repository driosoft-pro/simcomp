Sistema deComparendos

Plataforma distribuida para la gestión integral de infracciones de tránsito. Registra personas, vehículos e infracciones, y centraliza los comparendos en tiempo real.

// 01 — Arquitectura

Microservicios del Sistema

SERVICE — 01 / 04
👤
Personas Service
PORT :3001

Gestiona la información de los ciudadanos y sus licencias de conducción. Permite registrar, consultar y actualizar datos personales.

  • Registro de personas naturales
  • Gestión de licencias de conducción
  • Consulta por número de documento
  • Base de datos independiente (PostgreSQL)
SERVICE — 02 / 04
🚗
Vehículos Service
PORT :3002

Administra el registro de vehículos y sus propietarios. Permite registrar automotores y asociarlos a personas del sistema.

  • Registro de vehículos por placa
  • Asociación propietario-vehículo
  • Consulta por placa o propietario
  • Soporte múltiples tipos de vehículo
SERVICE — 03 / 04
📋
Infracciones Service
PORT :3003

Contiene el catálogo oficial de infracciones de tránsito con su código, descripción y valor de multa correspondiente.

  • Catálogo de infracciones por código
  • Valor de multa en pesos colombianos
  • Clasificación por tipo de infracción
  • Activación/desactivación de infracciones
SERVICE — 04 / 04
🚨
Comparendos Service
PORT :3004

Core del sistema. Registra los comparendos relacionando persona, vehículo e infracción. Gestiona estados y pagos.

  • Creación de comparendos por agentes
  • Estados: CREADO → PAGADO / ANULADO
  • Integración con los 3 servicios externos
  • Consulta y pago de comparendos

// 03 — Proceso

Flujo del Sistema

1
Registro de Persona
Un ciudadano o conductor es registrado en el sistema con sus datos personales e información de licencia de conducción.
→ personas-service :3001
2
Registro de Vehículo
El vehículo es registrado con su placa, marca, modelo y demás datos. Se asocia al propietario por su ID de persona.
→ vehiculos-service :3002
3
Configuración de Infracciones
El catálogo de infracciones es cargado con códigos oficiales, descripciones y valores de multa según normativa colombiana.
→ infracciones-service :3003
4
Creación del Comparendo
El agente de tránsito registra el comparendo. El servicio consulta a los otros 3 microservicios para validar persona, vehículo e infracción antes de persistir.
→ comparendos-service :3004 (consulta :3001, :3002, :3003)
5
Consulta y Pago
El ciudadano puede consultar sus comparendos activos y proceder al pago, cambiando el estado de CREADO a PAGADO.
→ comparendos-service :3004

Personas · :3001 👤 Gestión de personas y licencias

GET/api/personas/:idObtener persona por ID
GET/api/personas/documento/:numBuscar por número de documento
PUT/api/personas/:idActualizar datos de persona
POST/api/personas/:id/licenciasAgregar licencia de conducción
GET/api/personas/:id/licenciasListar licencias de una persona
Vehículos · :3002 🚗 Gestión de vehículos
GET/api/vehiculosListar todos los vehículos
POST/api/vehiculosRegistrar nuevo vehículo
GET/api/vehiculos/:idObtener vehículo por ID
GET/api/vehiculos/placa/:placaBuscar por placa
GET/api/vehiculos/propietario/:pIdVehículos por propietario
PUT/api/vehiculos/:idActualizar vehículo
Infracciones · :3003 📋 Catálogo de infracciones
GET/api/infraccionesListar todas las infracciones
POST/api/infraccionesCrear nueva infracción
GET/api/infracciones/:idObtener infracción por ID
GET/api/infracciones/codigo/:codBuscar por código oficial
PUT/api/infracciones/:idActualizar infracción
Comparendos · :3004 🚨 Core — Gestión de comparendos
GET/api/comparendosListar comparendos (con filtros)
POST/api/comparendosCrear nuevo comparendo
GET/api/comparendos/:idObtener comparendo detallado
GET/api/comparendos/numero/:numBuscar por número de comparendo
GET/api/comparendos/persona/:pIdComparendos de una persona
PATCH/api/comparendos/:id/pagarRegistrar pago del comparendo
PATCH/api/comparendos/:id/anularAnular comparendo

# Sistema de Comparendos de Tránsito
version: '3.8'

services:

  personas-service:
    build: ./personas-service
    ports: ["3001:3001"]
    environment:
      - DB_HOST=personas-db
    depends_on: [personas-db]

  vehiculos-service:
    build: ./vehiculos-service
    ports: ["3002:3002"]
    environment:
      - DB_HOST=vehiculos-db
    depends_on: [vehiculos-db]

  infracciones-service:
    build: ./infracciones-service
    ports: ["3003:3003"]
    environment:
      - DB_HOST=infracciones-db
    depends_on: [infracciones-db]

  comparendos-service:
    build: ./comparendos-service
    ports: ["3004:3004"]
    environment:
      - PERSONAS_URL=http://personas-service:3001
      - VEHICULOS_URL=http://vehiculos-service:3002
      - INFRACCIONES_URL=http://infracciones-service:3003
    depends_on: [comparendos-db, personas-service, vehiculos-service, infracciones-service]

  # --- PostgreSQL DBs independientes por microservicio ---
  personas-db:
    image: postgres:15
    environment: { POSTGRES_DB: personas_db }

  vehiculos-db:
    image: postgres:15
    environment: { POSTGRES_DB: vehiculos_db }

  infracciones-db:
    image: postgres:15
    environment: { POSTGRES_DB: infracciones_db }

  comparendos-db:
    image: postgres:15
    environment: { POSTGRES_DB: comparendos_db }
